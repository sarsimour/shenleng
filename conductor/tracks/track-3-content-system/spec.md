# Spec: Track 3 - Content System (CMS) & News Section

## 目标
建立基于 Payload CMS 的内容管理系统，支持发布行业文章和市场动态，通过独立的文章详情页提升 SEO 并支持跨平台引流。

## 关键需求
1. **Payload CMS 集成**:
   - 初始化 Payload CMS 3.0。
   - 定义 `Articles` 集合（标题、Slug、内容、摘要、封面图、发布时间）。
2. **首页新闻区块 (`NewsSection`)**:
   - 展示最新的 3 篇文章卡片。
   - 包含发布日期和简短摘要。
3. **文章详情页 (`/articles/[slug]`)**:
   - 响应式文章阅读页面。
   - 侧重排版美感，易于阅读。
   - 包含“获取解决方案”的侧边/底部 CTA，实现流量转化。
4. **SEO 支持**:
   - 动态生成文章的 Metadata。
   - 自动生成 Slug（基于标题或手动输入）。

## 验收标准
- 能够通过 `/admin` 后台发布文章。
- 首页新闻区块自动同步最新内容。
- 文章详情页能通过 URL 直接访问且样式正确。
