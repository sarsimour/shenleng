# Workflow: Conductor

## 开发原则
1. **Track-Based Development**: 所有工作都在明确的 Track 下进行。
2. **Spec & Plan First**: 在开始编码前，必须有明确的 Spec 和 Plan。
3. **Iterative Delivery**: 每个 Track 产生可交付的、经过测试的变更。

## 角色
- **Conductor**: 负责规划 Track、编写 Spec 和 Plan。
- **Agent**: 负责执行 Plan 中的任务。

## 流程
1. **Discovery**: 识别需求并记录在 `product.md`。
2. **Planning**: 创建 Track 并编写 `spec.md` 和 `plan.md`。
3. **Execution**: 执行任务，更新状态。
4. **Verification**: 验证更改并关闭 Track。
